---
title: "Get started with classicborne"
description: "Install classicborne, run a minimal Minecraft Classic server, then extend it with custom universe, player, and level classes to add your own game logic."
---

classicborne gets you from zero to a running Minecraft Classic server in a few minutes. This guide walks you through installation, a minimal working server, and the extension pattern you'll use for every custom feature you build. By the end you'll have a server that accepts connections and teleports players into a level, with a clear path to adding your own logic.

## Prerequisites

- Node.js 18 or higher
- npm, yarn, or pnpm

## Steps

<Steps>

<Step title="Install classicborne">

Add classicborne to your project:

<CodeGroup>

```bash npm
npm install classicborne
```

```bash yarn
yarn add classicborne
```

```bash pnpm
pnpm add classicborne
```

</CodeGroup>

classicborne is an ES module package (`"type": "module"`). Make sure your project's `package.json` also has `"type": "module"`, or use `.mjs` file extensions for your server files.

</Step>

<Step title="Create a minimal server">

The simplest possible classicborne server uses `BaseUniverse` directly with no subclassing.

```js server.mjs
import { BaseUniverse } from "classicborne/class/server/BaseUniverse.mjs"

const myUniverse = new BaseUniverse({
  port: 25565,
  postToMainServer: true,
})

// Flush block changes on all levels every 60 seconds.
setInterval(() => {
  myUniverse.levels.forEach(async (level) => {
    level = await level
    if (level.changeRecord) level.changeRecord.flushChanges()
  })
}, 60000)
```

`postToMainServer: true` posts a heartbeat to the ClassiCube server list so players can find your server. Set it to `false` during local development.

<Note>
  Always import from direct module paths like `classicborne/class/server/BaseUniverse.mjs`. The top-level `classicborne` barrel file is only used for TypeDoc generation and some of its re-exports may fail at runtime.
</Note>

</Step>

<Step title="Extend with custom classes">

The real power of classicborne is subclassing. Here is a complete example with a custom level, player, and universe wired together.

```js server.mjs
import { BaseLevel } from "classicborne/class/level/BaseLevel.mjs"
import { BasePlayer } from "classicborne/class/player/BasePlayer.mjs"
import { BaseUniverse } from "classicborne/class/server/BaseUniverse.mjs"

class MyLevel extends BaseLevel {
  constructor(...args) {
    super(...args)
    this.positionEventListeners = new Map()

    // Listen for player position updates while they are in this level.
    this.on("playerAdded", async (player) => {
      const onPosition = (position) => {
        console.log(position)
      }
      player.client.on("position", onPosition)
      this.positionEventListeners.set(player, onPosition)
    })

    // Clean up the listener when the player leaves.
    this.on("playerRemoved", async (player) => {
      const positionEventListener = this.positionEventListeners.get(player)
      player.client.removeListener("position", positionEventListener)
    })
  }

  static async teleportPlayer(player) {
    if (super.teleportPlayer(player) === false) return
    MyLevel.loadIntoUniverse(player.universe, "my-level").then(async (level) => {
      level.addPlayer(player, [60, 8, 4], [162, 254])
    })
  }
}

class MyPlayer extends BasePlayer {
  async initialize(...args) {
    const authenticated = await super.initialize(...args)
    if (!authenticated) return
    this.universe.addPlayer(this)
    MyLevel.teleportPlayer(this)
  }
}

class MyUniverse extends BaseUniverse {
  static playerClass = MyPlayer
}

new MyUniverse({
  port: 25565,
  postToMainServer: false,
})
```

The three key patterns here are:

- **`MyUniverse.playerClass`** — set this static property to control which class is instantiated for each new connection.
- **`MyPlayer.initialize`** — call `super.initialize` first to handle authentication, then register the player and teleport them.
- **`MyLevel.teleportPlayer`** — call `super.teleportPlayer` to remove the player from any current level, then load your level and add them to it.

<Tip>
  `super.teleportPlayer(player)` returns `false` if the player is already being teleported somewhere. Always check for this before proceeding.
</Tip>

</Step>

<Step title="Run your server">

```bash
node server.mjs
```

Your server will start listening on the port you configured. Connect to it with a Minecraft Classic client (such as ClassiCube) using `localhost:25565` for local development.

<Warning>
  classicborne creates a `./blockRecords/` directory next to your server file to store level persistence data. Make sure the process has write access to that directory, or use `NullChangeRecord` to opt out of persistence for a level.
</Warning>

</Step>

</Steps>
